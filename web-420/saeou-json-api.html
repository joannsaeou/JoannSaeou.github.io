What is JSON API?

JSON stands for JavaScript Object Notation -

It is an encoding scheme that is created to remove the need for a makeshift code for each application to communicate with servers that communicate in a defined way

How is JSON API differ from SOAP API?

SOAP is a standardized protocol that sends messages using other protocols such as a HTTP.

While the JSON API is a text-based data exchange format.

JSON Request Header
- The request header is for your app to detect what information was return and how it should handle the data.
- It is where the client sends JSON data to the browser and use the data right away
- A JSON request header contains a string used to authenticate requests.
JSON Request Body
- A request body is data sent by the client to your API.
- The HTTP method of POST, PUT, PATCH requests can have the JSON request body which is called body parameter.
JSON response body

A Response body is the data your API sends to the client.

The API will almost always has to send a response body because that is how they communicate with each other.

end-to-end data flow of a JSON API?

To use the data flow of JSON API you will start by importing model.json to create a new data flow

It is where you create a new workspace and add a data-flow of JSON API into it. The client can refresh the data-flow

and Users can start connecting from their data-flow

JSON API - document structure

- identified by media type

-Every request/response body must be underneath a single JSON object.

What does a format in JSON API appears as?

`/jsonapi/{entity_type}/{bundle}/{uuid}?_format=api_json`
